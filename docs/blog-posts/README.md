# Greenshot Blog Post Drafts

This directory contains draft blog posts about Greenshot releases and features.

## Purpose

Blog post drafts are created here to announce new features, improvements, and releases. These drafts can be used for:

- Publishing on the Greenshot website
- Social media announcements
- GitHub release notes
- Community newsletters

## Format

Blog posts should follow these guidelines:

### Structure
- **Title**: Clear, engaging headline about the feature/release
- **Metadata**: Date, version, and status (Development Build / Stable Release)
- **Introduction**: Brief overview of what's changed
- **Key Benefits**: User-focused description of improvements
- **Where to Find It**: Specific areas where users will see the changes
- **Technical Details** (optional): Brief technical information for interested readers
- **Call to Action**: How to try it out or where to download
- **Links**: References to changelog, downloads, and documentation

### Writing Style
- **User-friendly**: Focus on benefits, not implementation details
- **Natural language**: Avoid jargon and technical terms when possible
- **Positive tone**: Emphasize improvements and value
- **Clear structure**: Use headings, bullet points, and short paragraphs
- **Include links**: Make it easy to find more information

## Publishing Workflow

1. Draft the blog post in this directory
2. Review with the team
3. Coordinate with release timing
4. Publish to appropriate channels:
   - Greenshot website blog
   - GitHub release notes
   - Social media (Twitter, Reddit, etc.)
   - User forums and communities
